# 小書樓操作說明

> 版本：v2.0
> 最後更新：2026-07-19
> 正式入口：`https://admin.thesoword.com`

## 1. 一般發佈流程

日常發文優先使用管理後台，不需要直接登入 VM 或手動操作 Git。

1. 開啟 `https://admin.thesoword.com`，以後台帳號登入。
2. 新增或編輯作品、章節或公告。
3. 儲存後先檢查「待發佈變更」。
4. 填寫清楚的 commit message，選取要發佈的檔案並發佈。
5. 後台會將內容 commit、rebase 並推送到前台 `main`。
6. 到 GitHub Actions 確認 `Deploy 小書樓` 成功；通常數分鐘內可在正式站看到更新。

`author` 只能管理被指派的作品及其章節；`superadmin` 可管理所有內容、公告、帳號、作品歸屬與留言。

## 2. 內容規則

### 作品

- slug：小寫英數與連字號，例如 `the-unheard-melody`，建立後避免變更。
- `status`：`ongoing`、`completed`、`hiatus` 或 `draft`。
- `draft` 不會出現在讀者端。
- 摘要最多 200 字；封面放在 `soword/public/images/covers/`。
- 副標題為選填，最多 100 字；可在後台「作品 → 新增作品／編輯作品」填寫。
- 小書樓作品卡片以單行顯示副標題，作品詳情頁完整顯示；留空時不產生占位。

### 章節

- slug 建議：`{作品slug}-ch{三位數}`，例如 `the-unheard-melody-ch010`。
- `work` 必須填正確的作品 slug。
- `chapter` 使用正整數；建議以 10、20、30 排序，方便日後插章。
- 修改已發佈章節時填 `updatedAt`。

### 公告

- 填寫標題、發佈日期與內容。
- 重要公告可設 `pinned: true`。
- 公告只可由 `superadmin` 管理。

## 3. 留言審核

1. 登入管理後台並開啟留言管理。
2. 檢查 `pending` 留言的作品、章節、作者與內容。
3. 合宜內容設為 `approved`；不合宜內容設為 `rejected` 或刪除。
4. 只有 `approved` 留言會出現在章節頁。

讀者每 30 秒最多送出一則留言，每則最多 1000 字。不要直接在資料庫修改狀態，除非後台介面無法使用且已先備份。

## 4. 帳號管理

- 讀者自行以 Email／密碼註冊，需完成 Email 驗證。
- 忘記密碼由 Resend 寄送重設信。
- 後台管理者可建立 `author` 或 `superadmin`，並設定啟用狀態與作品歸屬。
- 小書樓不使用 Google 一鍵登入，也不需要申請 OAuth 憑證。

## 5. GitHub 備援發文

若後台介面暫時不可用，可直接在 GitHub 編輯 Markdown：

- 作品：`soword/src/content/works/{slug}/index.md`
- 章節：`soword/src/content/chapters/{slug}.md`
- 公告：`soword/src/content/announcements/{slug}.md`

送出前確認 frontmatter 符合 `soword/src/content/config.ts`，再 commit 到 `main`。GitHub Actions 會自動檢查與部署。這是備援流程；一般情況仍使用管理後台，才能同時保留稽核紀錄與 SQL 快照。

## 6. 開發與本地預覽

從 Windows 可直接 SSH 到 GCP，不需要開啟家用 VM。前台開發工作區：

```bash
cd /home/jasper/workspace/dev/thesowordcom
git status --short --branch
git pull --ff-only origin dev/gcp
cd soword
pnpm install --frozen-lockfile
pnpm run check
pnpm run build
pnpm run dev
```

瀏覽 `http://localhost:4321/soword/`。完成後先推 `dev/gcp` 驗證，再將同一變更帶入 `main`；不要直接在 GCP 的 `main` 備援工作區改檔。

## 7. 部署與驗收

前台 `main` 變更 `soword/**` 後會自動建置並部署到 S3／CloudFront。文件根目錄的純文件變更不會部署網站。

上線後檢查：

1. GitHub Actions 最新 `Deploy 小書樓` 為綠色。
2. `https://thesoword.com/soword/` 可開啟。
3. 新作品或章節可讀，搜尋能找到已發佈快照。
4. `https://admin.thesoword.com/api/health` 正常。
5. 小書樓導覽與頁尾沒有 RSS 按鈕或 GitHub 連結。

RSS feed 端點 `/soword/rss.xml` 仍供瀏覽器與搜尋工具自動探索，但不在頁面顯示按鈕。

## 8. 常見排錯

| 問題 | 處理方式 |
|---|---|
| 正式頁面沒更新 | 先確認 Actions 成功，再以 `Ctrl+Shift+R` 強制重整 |
| Actions 建置失敗 | 查看錯誤，多半是 frontmatter、slug 或 Markdown 格式不符 |
| 新作品不顯示 | 確認 `status` 不是 `draft` |
| 章節不顯示或順序錯 | 確認 `work` slug 正確且 `chapter` 不重複 |
| 登入、搜尋、收藏或留言失效 | 先測 `https://admin.thesoword.com/api/health` |
| 註冊信未收到 | 檢查垃圾郵件，再查後台 journal 與 Resend 狀態 |
| 留言沒出現 | 新留言預設待審核，需在後台核准 |
