# 小書樓操作說明

> 最後更新：2026-07-22

---

## 一、發布新章節（最常用）

**1. 拉取最新版本**
```bash
cd /home/jasper/workspace/thesowordcom
git pull origin main
```

**2. 建立章節檔案**

在 `soword/src/content/chapters/` 新增一個 `.md` 檔案。

檔名格式：`{作品slug}-ch{三位數}.md`

範例：`silent-echo-ch010.md`、`silent-echo-ch020.md`

**3. 填入內容**

```markdown
---
title: "第一章 潮聲"
work: "silent-echo"
chapter: 10
wordCount: 3842
publishedAt: 2026-06-02
---

正文從這裡開始……
```

| 欄位 | 說明 | 必填 |
|------|------|:----:|
| `title` | 章節標題（顯示用） | ✓ |
| `work` | 所屬作品的 slug | ✓ |
| `chapter` | 章節序號，建議用 10、20、30…（方便日後插章） | ✓ |
| `wordCount` | 本章字數（手填） | |
| `publishedAt` | 發布日期，格式 YYYY-MM-DD | ✓ |
| `updatedAt` | 修改日期（只有修改時才加，會顯示 NEW 標記 7 天） | |

**4. 提交並推送**
```bash
git add soword/src/content/chapters/silent-echo-ch010.md
git commit -m "feat(content): 新增《作品名》第一章「章節名」"
git push origin main
```

**5. 等待部署**

約 3–5 分鐘後，前往 https://thesoword.com/soword/ 確認章節出現。

可在 https://github.com/mayoruna06/thesowordcom/actions 即時查看進度。

## 二、新增整部作品

**1. 建立作品介紹頁**

新增資料夾：`soword/src/content/works/{作品slug}/`

在資料夾內建立 `index.md`：

```markdown
---
title: "沉默的回聲"
subtitle: "獻給仍在尋找歸途的你"
genre: "romance"
status: "ongoing"
summary: "退役偵探林硯試圖在海邊小城重建平靜生活……（200字以內）"
cover: "silent-echo.jpg"
tags:
  - "BL"
  - "現代"
  - "懸疑"
featured: false
publishedAt: 2026-06-02
---

## 關於這部作品

在這裡寫作品說明……

## 更新時間

每週日晚上十點更新。
```

**genre 可填值：** `romance`（言情）、`fantasy`（奇幻）、`sci-fi`（科幻）、`thriller`（懸疑）、`slice-of-life`（日常）、`other`

**status 可填值：** `ongoing`（連載中）、`completed`（完結）、`hiatus`（暫停）、`draft`（草稿，不對外顯示）

**subtitle 為選填副標題：** 最多 100 字。可直接在後台「作品 → 新增作品／編輯作品」填寫；小書樓作品卡片會以單行顯示，作品詳情頁則完整顯示。留空時不會產生空白區塊。

**tags 為作品分類標籤：** 可在後台用「、」或逗號分隔輸入。儲存時會自動清除多餘空白與重複標籤；每個標籤最多 30 字，一部作品最多 20 個。首頁、作品卡片與作品詳情頁的標籤均可點擊，進入該標籤的作品分類頁。

> 首頁會自動依狀態分成「連載中」和「已完結」兩區，右側側欄會自動列出排行和標籤。

**2. 放封面圖（選填）**

將封面圖放入 `soword/public/images/covers/`，檔名對應 `cover` 欄位的值。

建議尺寸：寬 400px、2:3 比例（如 400×600），JPEG，控制在 200KB 以內。無封面時會自動顯示書名卡片。

**3. 新增第一章**

同「一、發布新章節」步驟，`work` 填剛才建立的作品 slug。

**4. 推送**
```bash
git add soword/src/content/works/silent-echo/
git add soword/src/content/chapters/silent-echo-ch010.md
git add soword/public/images/covers/silent-echo.jpg  # 如果有封面
git commit -m "feat(works): 新增《沉默的回聲》+ 第一章"
git push origin main
```

## 三、新增公告（顯示在側邊欄）

在 `soword/src/content/announcements/` 新增 `.md` 檔案：

```markdown
---
title: "《沉默的回聲》開始連載"
publishedAt: 2026-06-02
pinned: false
---

公告內文（可省略）。
```

檔名建議：`2026-06-02-title.md`

---

## 四、修改已發布章節

**1. 開啟並修改檔案**

在 `soword/src/content/chapters/` 找到對應 `.md` 檔案，直接編輯。

**2. 更新 `updatedAt` 欄位**

```markdown
---
title: "第一章 潮聲"
work: "silent-echo"
chapter: 10
publishedAt: 2026-06-02
updatedAt: 2026-06-10    ← 改成今天日期，讀者端顯示 NEW 標記 7 天
---
```

**3. 推送**
```bash
git add soword/src/content/chapters/silent-echo-ch010.md
git commit -m "fix(content): 修正《沉默的回聲》第一章錯字"
git push origin main
```

## 五、用 GitHub 網頁直接發文（沒有電腦時）

1. 前往 https://github.com/mayoruna06/thesowordcom/tree/main/soword/src/content/chapters
2. 點右上角 **Add file → Create new file**
3. 檔名欄填入章節檔名，例如 `silent-echo-ch010.md`
4. 內容貼入完整 Markdown（含 frontmatter）
5. 滾到下方，Commit message 填：`feat(content): 新增第一章`
6. 選 **Commit directly to main** → 點 **Commit changes**

部署約 3–5 分鐘後自動上線。

## 七、Commit Message 格式

```
feat(content):   新增章節或作品
feat(works):     新增整部作品
fix(content):    修正錯字 / 排版
feat(announce):  新增公告
chore:           更新套件、維護
```

## 八、緊急排錯

| 問題 | 解法 |
|------|------|
| 部署後畫面沒更新 | 強制重整：Windows `Ctrl+Shift+R`，Mac `Cmd+Shift+R` |
| Actions 失敗（紅叉） | 前往 Actions 頁面看錯誤訊息，常見原因：frontmatter 格式錯誤 |
| 新作品不顯示 | 確認 `status` 不是 `draft` |
| 章節順序亂掉 | 確認同一作品的 `chapter` 數字沒有重複 |
| 「繼續閱讀」按鈕不出現 | 正常，需先讀過至少一章才會記住進度（存在瀏覽器 localStorage） |
| 想清除全站 CDN 快取 | Actions → Deploy 小書樓 → Run workflow → 勾選「是否清除全站快取」|
